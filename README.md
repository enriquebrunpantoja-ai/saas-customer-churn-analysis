# SaaS Customer Churn Analysis

## Project Overview

This project analyzes customer churn behavior for a SaaS company using data analytics and machine learning.

The objective is to identify behavioral patterns that lead customers to cancel their subscription and estimate the revenue at risk.

## Dataset

The dataset includes simulated SaaS customer information such as:

- Monthly subscription fee
- Customer tenure
- Number of support tickets
- Platform usage (logins per month)
- Churn status

## Tools Used

- Python
- Pandas
- Machine Learning
- Jupyter Notebook

## Key Findings

- Overall churn rate: 15.69%
- High-risk customers identified: 504
- Monthly revenue from high-risk customers: $21,500
- Revenue at risk: 5.12% of total revenue

## Business Insight

Customers with high support ticket activity and low platform engagement show significantly higher churn probability.

Identifying these customers early allows companies to implement retention strategies and reduce revenue loss.
